Expert consensus — themes raised independently by many of the panel

  • Color carries player identity, but nothing labels it (7 of 10). Every two-player chart, the Overview "Key Metrics" bars, and the Map outlines distinguish the two nations with a white vs. gold pairing and no legend anywhere. The app's core analytical payload — "which line is mine?" — is left to inference.
  • Red is spent on inert text (≈7 of 10). The Settings tab's labels and the game-detail date stamp use the alarm-red that's otherwise reserved for the delete action — reading as a warning where none exists.
  • Restated chrome pushes the data below the fold (Ive, Krug, Hall, Wroblewski, Garrett). The tall five-card summary repeats on every game-detail tab; tournament phase is stated three ways; the profile summary is duplicated between home and profile.
  • Unlabeled / icon-only controls (Norman, Nielsen, Krug, Wroblewski, Garrett). The owner toolbar — including a destructive Delete — is bare icons; the header search is an empty box; primary navigation hides behind a hamburger on a wide desktop.

Medium impact — inconsistency / unpolished, or 2+ experts

# Issue Expert(s) Screen(s) Suggested fix
5 Settings tab renders every field label and the Players table headers in red. "Map Size / Game Mode / Victory Conditions / DLC Enabled" and the table headers are saturated red — the alarm color — while equivalent labels on every other tab are muted tan. (Likely an accidental --color-brown token application.)
✓ verified — Settings tab
 Rams, Ive, Norman, Nielsen, Au, Garrett, Hall settings Recolor these labels and headers to the muted-tan label token used on the other tabs. Reserve red for destructive/error states only. Quick, high-consensus win.
6 Tournament grouping contradicts the cards' own badges, and phase is stated three times. Test 05 and Test 04 sit under the "ACTIVE" heading while their badge and Phase field both read "Setup". Each card also shows status in the section heading, a corner badge, and a "Phase" tile.
✓ verified — Tournaments list
 Rams, Ive, Norman, Nielsen, Garrett, Hall tournaments Derive group membership from the same status the badge shows (a Setup tournament belongs in a Setup/Draft group, not Active). Drop the redundant "Phase" tile since the badge already carries it, and reclaim that slot. Consider ordering by user relevance (tournaments you're in → active → open → past).
7 The game-detail summary strip is tall, and repeats on every tab. The five-card Player/Winner/Victory/Map/Turns band plus breadcrumb and tab bar consume ~290px before any tab content; on the Map and chart tabs the actual data opens below the fold. The five facts would fit one compact line.
✓ verified — repeats across game-detail tabs
 Ive, Krug, Hall, Wroblewski overview map Collapse the five cards into one slim metadata row (icon + value, thin dividers) so it stays as persistent context across tabs without owning the most valuable vertical space.
8 Primary navigation hides behind a hamburger on a wide desktop. Top-level destinations (Browse/Games, Tournaments, Account) aren't visible; the header has ample empty horizontal room. A signed-out visitor on the home page can't discover what sections exist without clicking the icon.
✓ verified — header on every screen
 Norman, Krug, Garrett anon home auth home Surface the core destinations as inline header links on desktop; keep the hamburger as the narrow-viewport fallback. The space between the wordmark and the Upload/search cluster is unused.
9 The game-detail date stamp is alarm-red (and low-contrast). The upload date (top-right, e.g. "2025-10-18") is the only red text on most tabs, grouped visually with the action icons though it isn't an action; small red on dark brown is also a weak-contrast pairing. The Map's "Turn" label is red too.
✓ verified — game-detail header, map
 Rams, Ive, Krug, Au, Levey overview map Render the date (and the Turn label) in the muted tan used for other secondary metadata, with an "Uploaded" caption; keep red strictly for destructive/critical affordances.
10 The Stats page stacks three rows of near-identical pills. Section tabs (Overview/Games/Stats), category tabs (Yields/Nations/Families/Laws/Cities/Tech), and chart-mode tabs (Per Turn / Cumulative / P25–P75 band / Game count) all share the same pill styling, so three nesting levels read as one undifferentiated wall of buttons.
✓ verified — Stats tab
 Ive, Krug, Wroblewski, Levey stats Differentiate the levels: keep the section nav as the strongest treatment, render the category level lighter, and make the chart-mode row a compact segmented control attached to the chart (or prefix a small "View:" label) so the hierarchy reads at a glance.
11 The header search is an empty, unlabeled box — and is absent for anonymous visitors. The signed-in search field shows only a magnifier, no placeholder, so its scope is unclear; the anonymous header drops search entirely, exactly for the audience doing public-game discovery.
✓ verified — anon vs auth headers
 Rams, Ive, Nielsen, Wroblewski, Krug, Hall, Levey auth home anon home Add placeholder text naming the scope ("Search games…") and a programmatic label; expose the search field to anonymous visitors scoped to public games.
12 Native OS form controls break the theme. The tournament "Open for signups" control is a bright-blue system checkbox; "Add a map…" is a native select; the games filter uses a white native date picker — all clashing with the styled tan controls used elsewhere (Filter / Columns / All games pills).
✓ verified — tournament-detail, games filter
 Au tournament-detail profile games Route checkboxes, selects, and date inputs through the existing styled control vocabulary; use the tan/gold accent for checkboxes rather than system blue.
13 Improvements table shows duplicate rows that read as a bug. "Baths" appears as three separate rows (totals 1, 2, 1) and "Courthouse" as three (3, 4, 2), with identical names and no distinguishing column to explain the repetition.
✓ verified — Improvements tab
 Ive improvements Aggregate same-named improvements into one row, or surface the distinguishing attribute (tier/level/era) as a visible column so the repetition is self-explanatory.
14 Summary stats use three densities, and the profile chips look like buttons. The same "at-a-glance" stats appear as large bordered cards (game-detail), tight chips inline in the profile breadcrumb, and a third density in the home sidebar. The profile chips share the raised pill look of the adjacent tab buttons but aren't interactive.
✓ verified — profile header vs game-detail cards
Au, Garrett, Wroblewski profile games overview Define one stat-tile component (and a single "compact" variant) and reuse it across profile header, home sidebar, and game-detail. Flatten the static stat chips so they don't read as clickable next to the real tab controls.
15 Small secondary labels are low-contrast. The muted-gray uppercase labels ("Player", "Winner", "Saves", "Win Rate", "Phase"…) sit only slightly above the card background — the lowest-contrast, smallest text on the page, and pervasive.
✓ verified — cards across home / profile / game-detail
Levey home overview Lift the label color to a warmer mid-tan that clears WCAG AA against the card surface; keep them visually secondary via size/weight rather than via crushed contrast.

What's already working

Several screens are the standard the rest should be pulled up to, and were repeatedly praised:

  • The user-profile games table — nation-colored names, Win/Loss pills, per-row chevron affordance, themed filters. (shot)
  • The full-size line charts (Victory Points, Science, Military) — clean grids, honest flat backgrounds — once a legend is added (finding #1).
  • Breadcrumbs are present and accurate on detail pages, giving good "where am I" support.
  • The permission gating reads correctly: anonymous viewers see the full analytics but not the owner toolbar.
