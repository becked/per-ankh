Design Language Audit

A static reference + audit of every distinct visual pattern found in the Per-Ankh cloud app (src/). Each pattern is rendered live alongside its file:line citations and class string. Where two usages differ in some specific way, variants are stacked side-by-side. The audit summary at the bottom collects the divergences worth consolidating.

Method: Tailwind Play CDN configured to mirror tailwind.config.js. Custom tokens (brown, tan, orange, blue-gray, gray-200, etc.) come from src/app.css :root. ECharts via CDN renders the chart section using the verbatim CHART_THEME from src/lib/config/charts.ts.

1. Color palette

The app declares 11 CSS variables in src/app.css:6-18; the Tailwind config exposes 9 of these as named tokens (dark-brown and border-gray are CSS-only). Beyond the tokens, 15+ raw hex values appear inline — most repeatedly. Tokens come first; inline hex follows with a usage count.

9. Audit summary — divergences to consolidate

Sorted roughly by impact (frequency × inconsistency cost). Pick a winner per row and we can do a follow-up sweep.

  1. Inline hex values that should be tokens. #35302B (15+ uses) and #2a2622 (7+ uses) are the two card backgrounds — adding --color-card-inner and --color-card-bg to app.css:6-18 would let every style="..." become a Tailwind class.
  2. Inline #DBDEE3 for body text (20+ uses). Visually equivalent to --color-gray-200 (#eeeeee). Either consolidate to text-gray-200 or add a --color-light-gray token. Hot spots: UploadModal.svelte, BulkUploadModal.svelte, GamePageSkeleton.svelte.
  3. Modal backdrop opacity differs. AboutModal.svelte:13 uses bg-black bg-opacity-50; ReimportButton.svelte:99 uses bg-black/70; BulkReparseModal.svelte:221 uses bg-black/70. Pick one.
  4. Error-text color split. text-orange (routes/login/+page.svelte:60, BulkUploadModal.svelte:445) vs text-red-400 (GameActions.svelte delete confirm, border-red-400). Recommend reserving red for destructive confirm and orange for everything else, or unify.
  5. Three SearchInput variants in one component. lib/SearchInput.svelte:17-30 — dark (#403A33 rounded-full py-1), light (white py-2 border-2 black), field (#201a13 py-2). Dark and field look very close; consolidate.
  6. Action popover width inconsistency. GameActions.svelte:180 = w-64; :268 and :351 = w-56. Same visual role.
  7. Card padding rule undocumented. p-3 vs p-4 — outer card always p-4, inner usually p-3, but tab pane uses p-8, modal uses p-6. Worth writing down or standardizing.
  8. Settings item gap variants (gap-1 vs gap-3). SettingsTab.svelte:27,33,49,59,69 — single-field uses gap-1, multi-field uses gap-3. Either document the rule or use the same gap.
  9. Table-header convention mismatch. SettingsTab.svelte = border-b-2 + base font; LawsTab.svelte:253 = border-b + uppercase tracking-wide. Pick one.
  10. One-off serif heading. BulkReparseModal.svelte:228 uses font-serif text-xl text-tan. Nothing else in the app uses serif — likely accidental.
  11. "Tan-ish" colors that aren't tan. Game list items use #c1872f / #f2a93b (CSS), context-menu Create button uses #ab9978 / #9a8a6c. None of these match --color-tan (#d2b48c).
  12. tan-hover token has no effect. app.css:11-12 defines --color-tan and --color-tan-hover as the same hex (#d2b48c). Tab triggers reference hover:bg-tan-hover — currently a no-op. Either pick a real hover shade or drop the variable.
  13. Unused tokens. --color-yellow (#ffff00) is defined and never referenced. --color-dark-brown and --color-border-gray are CSS-only (not Tailwind tokens) but referenced in CloudGameSidebar.svelte CSS.
  14. Skeleton bg typo. GamePageSkeleton.svelte:14 uses #36302a instead of #35302B. Almost certainly a typo.
