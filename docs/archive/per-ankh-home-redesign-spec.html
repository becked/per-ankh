Home page redesign — stats, tabs & search

Specification for restructuring the user home page (/users/[user_id]) into a tabbed layout with collection-scoped statistics, a two-search model, and removal of the persistent games sidebar.

01

Overview & scope

The user home page becomes a tabbed surface. A collection selector sits above the tab bar and scopes every tab. Statistics are computed and cached per collection. The persistent games sidebar is removed; the games list becomes a tab.

In scope for v1

  • Three-tab home layout: Overview, Stats, Games.
  • Collection selector as the scope control for all three tabs.
  • Per-collection cached stats aggregate.
  • Two distinct search affordances: a global header search and a local Games-tab search.
  • Removal of the persistent CloudGameSidebar from both the home page and game-detail.

Out of scope for v1

  • Save-search-as-collection.
  • Header search across other users' public corpora — v1 header search covers the signed-in user's own games only.
  • Per-collection public/private visibility as a data concept.
  • Standalone Timeline or Nations top-level tabs.
02

Page layout & tabs

Top to bottom: app header, collection scope row, tab bar, full-width tab content. No persistent side rail.

L1Header

Persistent across all tabs and across game-detail. Contains: brand/logo (links to the user's own home), Upload action, and the global header search input. Header search behavior is specified in section 04.

L2Scope row

A dedicated row directly below the header and directly above the tab bar. Contains the collection selector (left) and the save count for the selected collection. The selector's value scopes all three tabs. Specified in section 03.

L3Tab bar

Three tabs: Overview, Stats, Games. Overview is the default. The active tab is reflected in the URL (section 08). Tab content renders full-width below the bar.

  • Overview — summary metric tiles and the activity heatmap, scoped to the selected collection.
  • Stats — the full chart catalog with internal sub-navigation. Specified in section 06.
  • Games — the games list as a sortable, filterable table. Specified in section 05.
Tab content rule

Every tab reflects the collection selected in the scope row. Changing the collection re-renders all tab content. No tab has its own independent scope control.

03

Collection scope & caching

The collection selector is the single scope control. Its value is the cache key dimension for statistics.

Collection selector

S1Selector contents

The selector lists, in order:

  • All collections — the user's entire corpus. Default selection.
  • Public — a virtual filter: the subset of the user's games where is_public = true. This is a derived view, not a stored collection.
  • My collections — the user's owner-organizational collections, listed under a group label.

Each entry shows a count of games. is_public is a per-game field; there is no per-collection visibility concept and none is added.

S2Cache key

The stats aggregate is cached on the key (user_id, scope, collection_id), extending the existing cloud/src/stats/cache.ts model. scope distinguishes all / public / collection; collection_id is populated only when scope = collection.

S3Invalidation

A new save upload invalidates the all aggregate, the public aggregate if the save is public, and the aggregate for any collection the save is added to. Moving a save between collections invalidates both affected collection aggregates.

S4Switching collections

Selecting a collection triggers a single fetch of that collection's cached aggregate. The active tab re-renders from that one payload. Switching collections does not issue per-chart requests.

04

Search model

Two search affordances with separate jobs. Header search finds one game and navigates to it. Games-tab search narrows the games list in place.

Header search Games-tab search
Job Find one game, go to it Narrow the games list, view the set
Location App header, all tabs Games-tab filter row
Corpus All of the user's own games Games within the selected collection
Output Results dropdown Filtered table rows
On select goto the game's detail page n/a — table updates live
Collection scope Ignores it Respects it

Header search

H1Matching

Case-insensitive substring match against game name and nation, across all of the signed-in user's own games regardless of collection and regardless of is_public.

v1 corpus is the user's own games only. Other users' public games are not searched.

H2Results dropdown

Input opens a dropdown of matching games below it. Each row shows the game name (matched substring highlighted), the nation, the game date, and — for games in a collection — the collection name. Cap the list at a fixed count (e.g. 8) with the closest matches first.

H3Navigation

Clicking a result gotos that game's detail page (/games/[id]). Selecting a result does not change the collection selected in the scope row. Header search is purely navigational and never alters tab scope.

H4Empty & no-match states

Empty input: no dropdown. Input with no matches: dropdown shows a single "No games match" row. Matching is debounced.

Games-tab search

G-S1Free-text input

A free-text input in the Games-tab filter row, alongside the structured filter chips. Case-insensitive substring match against game name and nation, restricted to games in the currently selected collection.

G-S2Combination with chips

Free-text and structured chips (Nation, Result, Date) combine with AND. A row is shown only if it matches the free-text term and satisfies every active chip.

G-S3Output

Output is the filtered set of table rows, updated live. Games-tab search produces no dropdown and performs no navigation.

Visual distinction

The two inputs must not look interchangeable. Header search: placeholder "Find a game…", produces a dropdown. Games-tab search: placeholder "Filter these games…", sits in the filter row, produces table rows.

05

Games tab

The games list as a full-width sortable table with a free-text filter, structured chips, and a sort control. Absorbs the browsing role of the removed sidebar.

G1Table columns

Columns: Game name, Nation, Turns, Result, Date. Each column header is a sort control. One active sort at a time, ascending/descending toggle.

G2Filter row

Contains the free-text filter input (G-S1), structured chips for Nation, Result and Date, and a Sort control. Chips and free-text combine with AND (G-S2). The Sort control is independent of filtering.

G3Row navigation

Clicking a row navigates to that game's detail page. This is the path used to move between games now that the sidebar is removed (section 07).

G4Migrated sidebar machinery

The following move from CloudGameSidebar into the Games-tab table:

  • Rename / move-to-collection — the current right-click context menu becomes a per-row hover action menu.
  • Month dividers — rendered as group header rows, shown only when the active sort is by date.
  • Infinite scroll / pagination — retained for the table body.

The drag-to-resize width handle is not migrated; it has no purpose in a full-width tab.

G5List / grid toggle

A view toggle switches the table for a card-grid view of the same filtered, sorted set. Table is the default.

06

Stats tab

A single tab housing the full chart catalog. Internal sub-navigation moves between categories. All charts reflect the selected collection.

ST1Catalog & sub-nav

The Stats tab houses the existing ChartBundle catalog — 22 chart specs across 8 categories: Summary, Nations, Families, Rulers, Yields, Laws, Cities, Tech. An internal sub-navigation control selects the active category. Categories are not promoted to top-level tabs.

ST2Collection scoping

Every chart and metric in the tab is computed from the selected collection's cached aggregate (section 03). The existing ChartBundle is wired to the collection dimension; chart specs themselves are unchanged.

ST3Staleness indicator

When a tab is rendering a cached aggregate, show a small indicator (e.g. "cached aggregate") in the scope row, so a briefly-stale stat after a new upload is expected rather than surprising.

ST4Chart cross-filter

Clicking a data element in a chart (e.g. a nation bar) switches to the Games tab with the corresponding structured filter applied (e.g. nation=). This replaces the former chart-click-narrows-sidebar interaction. The chart click does not change the collection scope.

07

Sidebar removal

The persistent CloudGameSidebar is removed from both the home page and game-detail. No replacement rail is added.

SB1Home page

CloudGameSidebar is removed from /users/[user_id]. Its browsing function is served by the Games tab (section 05).

SB2Game-detail

CloudGameSidebar is removed from /games/[id]. To move between games, the user returns to the Games tab. Game-detail retains a back affordance to the Games tab. No pinned library list and no jump-to-game control is added in v1.

SB3Preserved capabilities

Rename, move-to-collection, month dividers and infinite scroll are preserved by migration into the Games-tab table (G4). The drag-to-resize handle is dropped.

08

Routing & URL state

The home page stays under /users/[user_id] so visitors can view other users' public corpora. Tab and scope state live in query parameters.

Parameter Values Meaning
tab overview · stats · games Active tab. Defaults to overview.
collection all · public · collection id Selected scope. Defaults to all.
category Stats sub-nav category Active Stats category. Applies when tab=stats.
nation, result, date, q filter values Games-tab chips and free-text. Apply when tab=games.
R1URL pattern

Tab state uses the query-parameter pattern /users/[user_id]?tab=stats, consistent with the existing game-detail ?tab= pattern. No top-level route rename.

R2Cross-filter target

A chart cross-filter click (ST4) navigates to ?tab=games with the relevant filter parameter set, e.g. ?tab=games&nation=Maurya.

R3Visitor view

When the viewer is not the owner, the home page renders the owner's public corpus. Header search (H1), upload, and per-row management actions (G4) are owner-only.

09

Build phases

A suggested sequence. Each phase is independently shippable.

1

Layout & navigation restructure

Header with collection scope row and tab bar. Overview and Games tabs. Games tab as a sortable table with chips, free-text filter and migrated sidebar machinery. Remove CloudGameSidebar from both surfaces. Routing per section 08.

2

Header search

Header search input, substring matching over the user's own games, results dropdown, navigation on select. Per section 04.

3

Collection-scoped stats

Extend the cache key with collection_id; wire invalidation. Stats tab with sub-nav over the existing ChartBundle, scoped to the selected collection. Staleness indicator.

4

Chart cross-filter

Chart data-element clicks navigate to the Games tab with the corresponding filter applied. Per ST4 and R2.
